# [LUMINA - Knowledge Engine] Hub Operating System V1

루미나의 기본 구조를 **Hub(설계/표준/투명 정산)** + **Spoke(실행/유통/현장 운영)** 로 고정한다.  
목표는 대표의 운영 부담을 줄이면서, 파트너 성과와 정산 신뢰를 동시에 높이는 것이다.

---

## 1) Operating Charter (운영 헌장)

### 1.1 Mission
- 루미나는 파트너가 실행 가능한 브랜딩/콘텐츠/정산 시스템을 제공한다.
- 루미나는 파트너와의 상생, 정산 투명성, 추적 가능한 기록을 최우선으로 한다.

### 1.2 Hub Responsibility (루미나 본부)
- 브랜드 로직/프롬프트/템플릿 설계
- 산출물 자동화 파이프라인 유지
- 정산 모델 및 API 표준 관리
- 품질 검수, 정책/보안/준법 통제

### 1.3 Spoke Responsibility (파트너 거점)
- 지역 실행(제조/유통/영업/운영)
- 입력 데이터 정확성 보장
- 현장 피드백 수집 및 루미나에 전달
- 고객/거래처 커뮤니케이션 실행

### 1.4 Non-Negotiables
- 숨은 수수료 UX 금지
- 근거 없는 효능/매출 과장 금지
- 정산은 line-item 단위로 공개
- 민감정보 최소 수집/최소 노출

## 2) Product Stack (허브 최소 기능 4종)

### 2.1 Copy Workspace Clone
- 파트너별 복제형 워크스페이스 생성
- 공통 코어는 유지, 파트너 변수만 주입

### 2.2 One-Click Output
- 입력(브랜드 정보/제품 정보/타겟) → 출력(카피/상세 초안/콘텐츠 초안)
- 승인 전 반드시 사람 검수 1회

### 2.3 Settlement Transparency
- 기간별, 항목별, 파트너별 정산 리포트
- `수익`, `수수료`, `공제`, `정산예정`, `정산완료` 상태 추적

### 2.4 SOP Console
- 온보딩, 검수, 게시, 리포팅을 체크리스트화
- 누가/언제/무엇을 처리했는지 기록

## 3) SOP (Standard Operating Procedure)

### [SOP-01] 파트너 온보딩 (D1-D3)
- 입력: 파트너 기본정보, 사업범위, 목표 KPI
- 출력: 파트너 복제 워크스페이스, 권한, 기본 템플릿
- 완료조건:
  - 관리자 계정 발급 완료
  - 초기 데이터 입력 검수 통과
  - 첫 산출물 1건 생성 완료

### [SOP-02] 산출물 생성/검수 (상시)
- 입력: 제품명, 핵심 포인트, 타겟 고객, 제약 조건
- 자동 출력: 상세페이지 초안/캠페인 카피/영상 스크립트 초안
- 검수 체크:
  - 과장 표현 여부
  - 브랜드 톤 일관성
  - 법/정책 리스크

### [SOP-03] 정산 마감 (주간/월간)
- 입력: 거래 로그, 수수료 규칙, 정산 주기
- 출력: line-item 리포트 + 합계 리포트
- 완료조건:
  - 이슈 항목 코멘트 기입
  - 파트너 확인 상태 기록
  - 지급/보류 상태 확정

### [SOP-04] 성과 리뷰 (월 1회)
- KPI 리뷰: 온보딩 시간, 재작업률, 파트너 유지율, 파트너 매출 기여
- 조치: 자동화 개선 1건 이상, 템플릿 개선 1건 이상

## 4) Pricing Template (요금제 템플릿)

### Plan A: Basic (월정액)
- 월 사용료: `₩____`
- 포함: 워크스페이스 1개, 산출물 월 `___`건, 정산 리포트 월 1회
- 초과 과금: 산출물 1건당 `₩____`

### Plan B: Growth (월정액 + 성과연동)
- 월 기본료: `₩____`
- 성과 연동: 순매출의 `__%`
- 포함: 워크스페이스 1개, 리포트 월 2회, 운영 코칭

### Plan C: Alliance (맞춤형)
- 다중 파트너/다중 브랜드 운영
- 커스텀 자동화, 전용 리포트, 전용 SLA

### 공통 계약 조항(필수)
- 정산 기준일/지급일 명시
- 환불/해지 조건 명시
- 데이터 소유권/사용권 범위 명시

## 5) KPI Dashboard Template

### 5.1 North Star
- `Partner Net Success Score (PNSS)` = 유지율 × 재구매율 × 정산신뢰지수

### 5.2 Core KPI
- 온보딩 리드타임 (목표: `__일 이하`)
- 산출물 재작업률 (목표: `__% 이하`)
- 파트너 월 유지율 (목표: `__% 이상`)
- 정산 이의제기율 (목표: `__% 이하`)
- 파트너 매출 기여율 (목표: `__% 이상`)

### 5.3 Weekly Review Format
- What happened: 숫자/사실
- Why happened: 원인 1~2개
- Action next: 다음 주 실행 1~3개
- Owner & Due: 담당자/기한

## 6) 14-Day Launch Plan (실행 캘린더)

### Day 1-3
- 허브 마스터 템플릿 고정
- 파트너 1곳 데이터 수집
- 첫 자동 산출물 검수

### Day 4-7
- 파일럿 스포크 1곳 오픈
- 정산 포맷 테스트
- 피드백 반영

### Day 8-10
- 대시보드 지표 연결
- 운영 병목 1차 제거

### Day 11-14
- 요금제/계약 확정
- 2번째 스포크 복제 배포

## 7) 역할 배치 (최소 인력)

- Hub Owner (대표): 방향/우선순위/최종 승인
- Operator (1명): 일일 운영, 체크리스트, 커뮤니케이션
- Builder (1명, 파트타임 가능): 자동화/버그 수정/리포트 연결
- AI Copilot: 초안 생성/반복 작업 자동화
